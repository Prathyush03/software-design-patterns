Master Table — 28 Patterns, 3 Rules

Every pattern in Grokking the Coding Interview Patterns, audited against the same three rules derived in Chapter 1 and reused ever after. Click any pattern name to open its full interactive chapter. The rules never change — only their implementation does.

Rule 1 — Name the invariant
The property that is true after every single step. The invariant IS the pattern.
Rule 2 — Define the move
One deterministic step: irreversible progress + a one-line safety proof. The proof doubles as the complexity bound.
Rule 3 — Mark the harvest
The exact moment an answer is visible. Grab it before the move destroys the evidence.
How to read this table: each row states how one pattern implements the three rules. Click a row to expand its variations — within a pattern, variations keep the rules and change only the implementation (usually the move's proof or the harvest's placement). The Signal column is the recognition cue: what a problem statement sounds like when this pattern applies.
